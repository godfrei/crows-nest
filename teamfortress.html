Custom Resourse Downloading
[00:10 PST - By DieHard]
Valve Software asked me if I could pass this around. So I figure what the heck, they made TFC, so I guess we could do this favor for them. =)
Here's the info on the new map downloading feature. It would be great if map makers packaged their maps in this fashion so server operators could set it up painlessly.

CUSTOM RESOURCE DOWNLOADING
Something that has always limited the propagation of custom maps has been the lack of support for custom map resources. Running a custom map that has it's own .wad file, custom sprites, or precached sounds has been something of a burden to server operators. So, we've added a new feature/function to help simplify the process.

For this example let's assume we have a Team Fortress map called "mymap.bsp". This map uses a few different custom resources. It has it's own .wad file called "mytextures.wad", a new model called "mymodel.mdl", and a new sprite called "mysprite.spr". As a server operator you will need to go through these steps to allow these resources to be downloaded:
  1. Create a file called "mymap.res" and place it into the \tfc\maps directory. The .res file must be the same as the .bsp name.
  2. Add the resource names relative to the game directory in this file. The file should look like this:
    mywad.wad
    models\mymodel.mdl
    sprites\mysprite.spr
  3. Run your server with sv_allowdownload set to 1.
When a client connects they will start downloading these files. The client's download speed/bandwidth is throttled according to their rate. This will not replace any files that are currently on the client's machine.
By using this new format more players will be able to download complete maps, and they no longer have to worry if the map has customized files. I know valve has included great maps in their patches, but there are a lot of great user made maps. Hopefully with this change more players will start playing user made maps.

Interview and Stuff
[09:08 PST - By DieHard]
TF2 Bunker scored an interview with Dr. Steven Collins, Havok's Chief Technical Officer. If you are unfamiliar with Havok, it is a game physics engine, unofficially this physics engine will be used Half-Life 2. According to the questions answered in the interview, this engine could be used for players clothes, trees, water ripples, etc. Basically what we are looking at is a 100% interactive environment. Dr. Collins answered one question saying this:
For a game that has been fully designed in advance (without consideration of the physics) this technology is mostly used to add special effects like realistic water, great explosions, debris, perhaps character clothing etc. Things really get interesting however when physics is in the game design from day one. In this case, the design of the game may be significantly altered: now designers can create fully interactive environments, where all the objects are physical and be used in any way imaginable. This completely opens up the gameplay becoming a really non-linear experience. Perhaps your goal in a game is to jam a gate locking mechanism using a spanner; imagine you had lost the spanner but had just blown a branch off a tree during a previous battle. With a real physics engine, that branch could be used in place of the spanner with no extra coding!
Now that is slick!
